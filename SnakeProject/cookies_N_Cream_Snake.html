Tech Stack

Notable Code Features

üéÆ Input Buffering: The future_Turns queue system allows players to buffer up to 2 direction changes, enabling precise control during fast gameplay.

üìä 7-Segment Display: Each number is represented by a 7-bit binary pattern (e.g., '1111110' for 0), with individual segments colored based on whether they should be lit.

üç¶ Distance-Based Collision: Uses snake_Turtles[0].distance(cookies_N_Creme) to detect when the snake eats ice cream, which is more reliable than exact coordinate matching.

üí≠ Development Notes

Real talk: I procrastinated a little too much on this one. The game is playable and fun, but there are definitely some rough edges I didn't get to polish before the deadline.

What's Working ‚úÖ

  • Core Gameplay: The snake moves, eats ice cream, grows, and the scoring system works perfectly
  • Custom Graphics: All the sprites look great - snake head, body, and cookies & cream ice cream
  • 7-Segment Displays: Both current score and high score track properly with LED-style digits

What's Not Finished ‚ö†Ô∏è

  • Death System: The snake can't actually die yet - no collision detection with walls or itself
  • Respawning: The restart function exists but isn't fully wired up
  • Ice Cream Spawning: Sometimes spawns inside the snake because I commented out the coordinate tracking that prevents that (it was causing crashes due to floating-point errors)

The Coordinate Problem ü§Ø

This was the biggest headache. I tried to track which tiles the snake occupies so ice cream wouldn't spawn on top of it, but Python's floating-point math kept breaking everything:

  • Negative Zero: Sometimes coordinates would be -0.0 instead of 0.0, which looks the same but Python treats as different when checking lists
  • Rounding Errors: Moving the snake 1 pixel at a time for 100 pixels doesn't always equal exactly 100 pixels due to floating-point precision. After a few moves, coordinates would be off by 0.00000000000024 and list lookups would fail
  • The Solution I Didn't Have Time For: I needed to round all coordinates more aggressively or switch to using integer pixel positions, but ran out of time debugging it

AI Assist ü§ñ

I used AI (Claude) for two main things:

  • Pillow/PIL: Had no idea how to use it before this, so AI helped me figure out how to resize images and convert them to Turtle-compatible formats
  • Debugging: When I got weird errors (especially with the floating-point coordinate stuff), AI helped me understand what was going wrong and suggested fixes

Despite the unfinished parts, I'm pretty proud of what I got working. The game is fun to play (until you realize you can't lose), and building those 7-segment displays from scratch was super satisfying!
